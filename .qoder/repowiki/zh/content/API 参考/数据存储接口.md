## 简介
本文件面向 ImgPrompt 扩展的数据存储接口，系统性梳理本地存储的数据结构设计、键名规范、序列化与反序列化处理、访问模式与并发控制、清理与归档策略、迁移与升级方案，并提供可直接定位到源码的示例路径与性能优化建议。目标是帮助开发者与使用者准确理解与高效维护本地数据层。

**更新** 本次重大变更从 Chrome 存储 API 迁移到 IndexedDB API，提供了更强大、更灵活的数据存储解决方案。

## 项目结构与数据存储概览
- 存储区域：扩展使用 IndexedDB 作为主要存储引擎，同时保留部分 Chrome Extension Storage API 的使用场景，确保向后兼容性。
- 数据库配置：
  - 数据库名称：ImgPromptDB
  - 版本：1
  - 对象存储：history（历史记录存储）
  - 索引：timestamp（时间戳索引，支持降序查询）
- 关键脚本：
  - background.js：服务工作线程，负责数据库初始化、历史记录的增删查、客户端 ID 生成、分析事件上报等。
  - content.js：内容脚本，负责与页面交互、面板状态管理、监听存储变更、发送消息给后台等。
  - options.js：选项页脚本，负责设置表单、自定义模板、历史记录展示与操作、自动保存与通知等。
  - config.js：共享配置，包含默认设置、UI 文案、错误码、分析配置键名等。
  - manifest.json：声明权限与能力，明确 storage 和 sidePanel 权限。
  - options.html：选项页界面，承载设置与历史记录展示。
- 数据流：设置项通过 chrome.storage.local 存储，历史记录通过 IndexedDB 存储，在后台、内容脚本、选项页之间同步。

## 核心数据模型
- 客户端 ID（client_id）
  - 用途：唯一标识用户设备，用于分析事件去重与关联。
  - 存储位置：chrome.storage.local
  - 键名：clientId
  - 写入时机：首次访问时生成并写入本地存储。
  - 示例路径：[ensureClientId:357-368](file://background.js#L357-L368)

- 设置配置（settings）
  - 用途：扩展运行所需的核心参数，如 API 地址、模型、密钥、语言、最大图像边长、是否启用悬浮按钮等。
  - 存储位置：chrome.storage.local
  - 默认值：DEFAULT_SETTINGS
  - 读取合并：启动时从本地存储读取，缺失字段用默认值补齐。
  - 示例路径：[loadSettings:349-355](file://background.js#L349-L355)、[DEFAULT_SETTINGS:5-21](file://config.js#L5-L21)

- 历史记录（promptHistory）
  - 用途：保存每次生成的提示词结果、触发来源、时间戳、图片数据等。
  - 存储位置：IndexedDB（ImgPromptDB.history）
  - 结构要点：对象存储，每条记录包含 id、timestamp、prompts（zh/en）、srcUrl、imageDataUrl、pageUrl、model、trigger 等。
  - 索引：timestamp（降序，支持高效查询）
  - 长度限制：MAX_HISTORY_ITEMS，默认 50。
  - 示例路径：[saveToHistory:439-487](file://background.js#L439-L487)、[getHistory:489-519](file://background.js#L489-L519)、[deleteHistoryItem:521-538](file://background.js#L521-L538)、[clearHistory:540-557](file://background.js#L540-L557)

- 自定义模板（customTemplates）
  - 用途：用户自定义的 userPrompt 模板集合。
  - 存储位置：chrome.storage.local
  - 结构要点：对象映射，键为模板 id，值含 title、prompt。
  - 示例路径：[options.js 中的自定义模板读写:119-149](file://options.js#L119-L149)、[options.js 初始化渲染:182-217](file://options.js#L182-L217)

- 分析配置键（analyticsConfig）
  - 用途：控制是否允许分析事件上报。
  - 存储位置：chrome.storage.local
  - 键名：analyticsConfig
  - 示例路径：[safeTrackAnalyticsEvent:431-437](file://background.js#L431-L437)、[config.js:268](file://config.js#L268)

## 并发控制与事务处理
- 浏览器存储并发：
  - chrome.storage.local 提供原子性写入，但跨脚本并发仍需谨慎。建议：
    - 单一入口写入：将写入集中在后台或选项页，避免多处同时写入。
    - 批量写入：一次 set 多个键，减少多次写入带来的竞争。
    - 读写顺序：先读后写，避免竞态条件。
- IndexedDB 事务处理：
  - 使用显式事务确保数据一致性，支持读写和只读两种模式。
  - 支持多对象存储的复合事务。
  - 示例路径：[getHistory 事务:489-519](file://background.js#L489-L519)、[saveToHistory 事务:439-487](file://background.js#L439-L487)
- 请求级并发：
  - activeRequests Map 记录进行中的生成请求，支持取消与去重。
  - 示例路径：[activeRequests 管理:17-132](file://background.js#L17-L132)
- UI 并发：
  - content.js 通过 isGenerating、activeRequestId 等状态变量避免重复生成与状态错乱。
  - 示例路径：[content.js 状态变量:36-54](file://content.js#L36-L54)

## 结论
